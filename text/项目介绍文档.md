# 灵山胜境 · AI 智能导游（灵小导）— 项目介绍

> **第十五届中国软件杯大赛 A 组赛题 A5 — 景区导览服务 AI 数字人**
>
> 出题企业：锐捷网络（苏州）有限公司

---

## 一、项目概述

"灵小导"是一个面向智慧景区的 **AI 数字人智能导游系统**，以无锡灵山胜境为示范景区，利用大语言模型（LLM）、检索增强生成（RAG）、语音交互和数字人技术，为游客提供 7×24 小时在线智能导览服务，同时为景区管理者提供数据大屏与游客洞察。

### 一句话说明

> 游客用语音或文字向 AI 数字人导游提问，系统基于景区知识库实时生成专业、亲切的回答，并驱动数字人以语音和表情进行互动回应——所有服务全天候在线，无需预约。

## 二、核心功能

### 🎙️ 游客端

| 功能 | 说明 |
|------|------|
| **智能问答** | 基于灵山胜境知识库（2 份文档 / 37 个知识块），回答游客关于景点、历史、文化、游览路线等各类问题 |
| **流式打字机效果** | AI 回答逐字实时显示，感知延迟极低，交互体验流畅 |
| **语音输入** | 浏览器端 Web Speech API 语音识别，支持中文语音转文字输入 |
| **语音合成播放** | edge-tts 生成自然语音，5 种中文发音人可选，自动播放 + 手动重播 |
| **个性化路线推荐** | 三合一兴趣卡片——历史文化深度路线、自然风光爱好者路线、亲子家庭欢乐路线，每条路线附带景点时间线与实用贴士 |
| **多轮对话记忆** | 支持上下文理解与代词消解（如"它有多高？"→ 自动关联上一轮提到的灵山大佛） |
| **CSS 2D 数字人** | 曼荼罗光环 + 8 瓣金莲 + 浮动灵光粒子，5 种表情 + 口型动画 + 呼吸/眨眼/说话微动，CSS 变量主题系统，零外部依赖 |
| **VTube Studio 集成** | 可选接入 Live2D/VRM 3D 模型，WebSocket 实时控制口型、表情和动作 |

### 📊 管理后台

| 功能 | 说明 |
|------|------|
| **数据大屏** | 6 统计卡片 + ECharts 交互趋势图 + 热门问题排行 + 时段热力图，30 秒自动刷新 |
| **知识库管理** | 拖拽上传文档（txt/pdf/docx），自动解析、分块、建立 TF-IDF 索引，支持查看和删除 |
| **交互记录** | 分页查看全部问答记录，支持关键词搜索、日期筛选、详情弹窗 |
| **游客报告** | 基于交互日志自动生成分析报告：关注点分布、趋势变化、热门问题 Top 10、改进建议 |
| **数字人配置** | VTube Studio 连接管理、表情/动作测试面板、角色名称/欢迎语/发音人设置 |

### 技术选型一览

| 层面 | 技术 | 选型理由 |
|------|------|---------|
| 前端 | Vue 3 + Vite + Element Plus | 轻量、中文生态好、移动端响应式 |
| 图表 | ECharts 5 | 国产、文档完善、管理后台标配 |
| 语音识别 | Web Speech API (浏览器) | 零成本、Chrome/Edge 内置、中文识别率高 |
| 语音合成 | edge-tts | 免费、中文质量极高、40+ 发音人 |
| 大语言模型 | DeepSeek-chat | 性价比高、中文能力强、支持流式生成 |
| 向量检索 | sklearn TF-IDF (char ngram 1-3) | 离线可用、无需 GPU、竞赛环境友好 |
| 数字人 | CSS 2D（曼荼罗/莲花/粒子/14 种动画/5 种情绪）+ VTube Studio | CSS 保底可用，VTS 3D 提升表现力 |
| 数据库 | SQLite | Python 内置、零配置、竞赛场景足够 |

## 四、数据流（以一次游客问答为例）

```
游客说"灵山大佛有多高？"
    │
    ▼
Web Speech API → 文字识别
    │
    ▼
POST /api/chat/ask/stream ──→ TF-IDF 检索 top-5 相关文档块
    │                              │
    │                              ▼
    │                    拼接 System Prompt + 知识上下文 + 对话历史
    │                              │
    │                              ▼
    │                     DeepSeek-chat 流式生成
    │                              │
    ▼                              ▼
SSE 逐 token 推送 ←────────── 每个 token
    │
    ▼
前端打字机显示 ──→ 完成后 edge-tts 合成语音 ──→ 播放
    │
    ▼
情绪关键词检测 ──→ CSS 数字人表情切换 / VTS 表情联动
    │
    ▼
交互日志写入 SQLite analytics.db (问题、回答、来源、耗时、情绪)
```

## 五、创新亮点

| 维度 | 创新点 | 说明 |
|------|--------|------|
| **交互体验** | 流式打字机 + TTS 全链路 | SSE 逐 token 推送 → 前端异步渲染 → edge-tts 合成播放，延迟感知极低 |
| **智能程度** | 多轮对话 + 代词消解 | 保留最多 6 轮历史，支持"它""这个""那里"等指代理解 |
| **数字人** | 双轨方案 | CSS 2D 保底（零依赖即可运行）+ VTube Studio 3D 增强（WebSocket 实时控制） |
| **情绪联动** | LLM 输出 → 表情映射 | 自动检测回答中的情绪关键词，驱动数字人表情同步切换 |
| **场景覆盖** | 游客端 + 管理端闭环 | 不仅服务游客问答，还提供数据大屏、交互记录、游客报告、知识库管理等完整管理能力 |
| **工程实践** | 零新增硬件 | 纯 Python 生态，SQLite + JSON 文件存储，无需额外数据库或 GPU |
| **语音链路** | 全免费方案 | 浏览器 ASR → RAG+LLM → edge-tts TTS，无需付费语音 API |
| **离线可用** | TF-IDF 替代向量数据库 | 针对 HuggingFace 不可达的竞赛环境，sklearn char-ngram TF-IDF 纯离线运行 |

## 七、快速体验

### 环境要求

- Python 3.10+、Node.js 18+
- DeepSeek API Key（从 [platform.deepseek.com](https://platform.deepseek.com) 获取）
- Chrome / Edge 浏览器

### 三步启动

```bash
# 1. 启动后端
cd backend
pip install -r requirements.txt
# 编辑 .env 填入 DEEPSEEK_API_KEY
python -m uvicorn main:app --host 0.0.0.0 --port 8000

# 2. 启动前端
cd frontend
npm install
npx vite

# 3. 浏览器访问
# 游客端 → http://localhost:5173/visitor
# 管理后台 → http://localhost:5173/admin
```

> 详细部署说明请参阅 [产品部署和使用手册.md](产品部署和使用手册.md)
