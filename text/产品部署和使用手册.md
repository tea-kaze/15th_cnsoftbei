## 二、快速部署

### 2.1 获取代码

```bash
# 将项目复制到本地
# 项目根目录: A5/
cd A5
```

### 2.2 后端部署

```bash
# 1. 进入后端目录
cd backend

# 2. 安装 Python 依赖
pip install -r requirements.txt

# 3. 配置 DeepSeek API Key
# 编辑 .env 文件，填入你的 API Key:
#   DEEPSEEK_API_KEY=sk-your-key-here
#   DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
#   DEEPSEEK_MODEL=deepseek-chat

# 4. 确认知识库索引存在
# 文件 backend/knowledge_index.json 应包含 37 个 chunks
# 如果不存在或需要重建，通过管理后台上传文档

# 5. 启动后端服务
python -m uvicorn main:app --host 0.0.0.0 --port 8000
```

启动成功标志：
```
INFO:     Started server process [xxxxx]
INFO:     Uvicorn running on http://0.0.0.0:8000
```

验证：浏览器访问 `http://localhost:8000/api/health`，应返回 `{"status":"ok"}`

### 2.3 前端部署

```bash
# 1. 进入前端目录
cd frontend

# 2. 安装 Node.js 依赖
npm install

# 3. 开发模式启动
npx vite

# 4. 生产构建（可选）
npx vite build
# 构建产物在 dist/ 目录，可部署到任意静态服务器
```

启动成功标志：
```
VITE v8.0.12  ready in xxx ms
➜  Local:   http://localhost:5173/
```

### 2.4 访问系统

| 地址 | 页面 | 说明 |
|------|------|------|
| `http://localhost:5173/visitor` | 游客端 | 聊天 + 路线推荐 |
| `http://localhost:5173/admin` | 管理后台 | 数据大屏 / 知识库 / 报告 |

> **注意**: 前端 Vite 开发服务器已配置代理，`/api` 请求自动转发到后端 `http://localhost:8000`。

## 三、配置说明

### 3.1 后端配置 (`backend/.env`)

```env
# DeepSeek API（必填）
DEEPSEEK_API_KEY=sk-your-key-here        # 从 platform.deepseek.com 获取
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
DEEPSEEK_MODEL=deepseek-chat
```

**获取 API Key**:
1. 访问 https://platform.deepseek.com
2. 注册/登录账号
3. 在 API Keys 页面创建新 Key
4. 复制 Key 填入 `.env` 文件

### 3.2 前端配置 (`frontend/vite.config.js`)

```js
export default defineConfig({
  server: {
    port: 5173,              // 前端开发端口
    proxy: {
      '/api': {
        target: 'http://localhost:8000',  // 后端地址
        changeOrigin: true,
      },
    },
  },
})
```

## 四、使用指南

### 4.1 游客端（`/visitor`）

#### 文字问答
1. 在底部输入框输入问题（如"灵山大佛有多高？"）
2. 按回车或点击发送按钮
3. 观察 AI 回答逐字显示（流式打字机效果）
4. 回答完成后自动合成语音播放

#### 语音输入
1. 点击输入框左侧的 🎤 麦克风按钮
2. 浏览器会请求麦克风权限，点击允许
3. 开始说话（中文），按钮变红表示录音中
4. 说完后点击按钮停止录音，自动发送识别结果

> **提示**: 语音识别使用浏览器内置的 Web Speech API，Chrome/Edge 支持最佳。如果浏览器不支持，会自动降级为纯文字输入。

#### 路线推荐
1. 点击聊天区上方的「🗺️ 路线推荐」按钮
2. 点击三张兴趣卡片之一（历史文化/自然风光/亲子家庭）
3. 或手动输入兴趣描述后点击推荐
4. 查看推荐路线的景点时间线、实用贴士
5. 点击追问按钮继续了解路线详情

#### 语音重播
- 每条 AI 回答右下角有 🔊 按钮，点击可重新播放语音

#### PC/移动端布局切换
- 头部右上角有 💻/📱 切换按钮
- **移动端模式**：竖屏布局（max-width: 480px），数字人在顶部，聊天区在下方
- **PC 端模式**：横屏布局（CSS Grid），左侧 35% 面板放数字人（放大）+ 工具栏，右侧 65% 面板放聊天区
- 切换按钮点击后立即生效，布局偏好自动保存到浏览器（localStorage），刷新页面后保持选择

### 4.2 管理后台（`/admin`）

#### 数据大屏
- 顶部 6 个统计卡片：今日/本周服务人次、累计问答、平均响应时间、知识文档数、今日独立访客
- 交互趋势折线图（近 30 天）
- 热门问题柱状图
- 今日时段分布柱状图
- 每 30 秒自动刷新

#### 交互记录
- 分页表格查看所有问答记录
- 搜索框支持关键词搜索
- 日期范围筛选
- 点击眼睛图标查看完整问答详情和检索来源

#### 知识库管理
- 拖拽或点击上传文档（支持 .txt / .pdf / .docx）
- 文档列表显示文件名和知识块数
- 点击删除清理旧文档

#### 游客报告
- 选择分析周期（7/14/30 天）
- 点击生成报告
- 查看游客关注点分析、交互量趋势、热门问题 Top 10
- 查看 AI 服务改进建议

#### 数字人配置
- 查看 VTube Studio 连接状态
- 点击「连接 VTube Studio」建立连接
- 测试表情（开心/悲伤/惊讶/愤怒/中性）
- 查看 VTS 状态（模型名称/FPS/可用表情/运行时长）
- 测试动作（挥手/鞠躬/思考/完整欢迎）
- 设置角色名称、欢迎语、默认 TTS 发音人

## 五、知识库管理

### 5.1 上传新文档

1. 访问管理后台 → 知识库管理
2. 将文档拖入上传区，或点击选择文件
3. 支持格式：.txt / .pdf / .docx
4. 上传成功后自动解析、分块、建立索引

### 5.2 文档格式建议

| 格式 | 适用场景 | 注意 |
|------|---------|------|
| .txt | 纯文本资料 | UTF-8 编码 |
| .pdf | 扫描件/排版文档 | 仅提取文字，图片无法索引 |
| .docx | Word 文档（推荐） | 支持表格内容提取 |

### 5.3 检索效果优化建议

- 文档内容以Q&A格式组织（一问一答），检索命中率更高
- 每个自然段控制在 200-500 字，利于分块
- 包含关键数字、名称、日期等事实性信息
- 表格数据会被保留并以 `|` 分隔提取

## 六、VTube Studio 集成（可选）

### 6.1 安装步骤

1. **安装 VTube Studio**：从 Steam 购买（约 ¥40）或在 iOS App Store 免费下载
2. **准备模型**：使用 VRoid Studio（免费）制作 VRM 3D 角色，或从 VRoid Hub/Booth 下载现成模型
3. **导入模型**：将 .vrm 文件拖入 VTube Studio
4. **开启 API**：VTube Studio 设置 → 勾选「Start API」→ 端口保持 8001

### 6.2 连接系统

1. 启动 VTube Studio 并加载模型
2. 访问管理后台 → 数字人配置
3. 点击「连接 VTube Studio」
4. 首次连接时 VTube Studio 会弹出确认对话框，点击「允许」
5. 连接成功后状态显示「VTS 在线」

### 6.3 配置 OBS Virtual Cam（网页嵌入）

1. 安装 OBS Studio
2. 添加「窗口采集」→ 选择 VTube Studio 窗口
3. 右键窗口采集 → 滤镜 → 色度键（绿幕抠除）或直接设置透明背景
4. 工具 → 开启 Virtual Cam
5. 游客端 → 点击数字人头像旁的 📷 图标切换到 Virtual Cam 模式

### 6.4 VTube Studio API 端点

| API | 说明 |
|-----|------|
| `GET /api/vts/status` | 连接状态 |
| `POST /api/vts/connect` | 手动连接 |
| `POST /api/vts/expression` | 表情控制 |
| `POST /api/vts/speak` | 口型同步 |
| `POST /api/vts/hotkey` | 动作触发 |
| `POST /api/vts/greet` | 欢迎流程 |

## 七、故障排除

### 7.1 后端启动失败

| 症状 | 原因 | 解决 |
|------|------|------|
| `ModuleNotFoundError: fastapi` | 依赖未安装 | `pip install -r requirements.txt` |
| `端口 8000 被占用` | 其他程序使用中 | 修改启动端口或关闭占用程序 |
| `.env 文件找不到` | 未配置 API Key | 复制 `.env.example` 为 `.env` 并填入 Key |

### 7.2 前端启动失败

| 症状 | 原因 | 解决 |
|------|------|------|
| `npm: command not found` | Node.js 未安装 | 安装 Node.js 18+ |
| `Cannot find module` | 依赖未安装 | `cd frontend && npm install` |
| `端口 5173 被占用` | 其他程序使用中 | 修改 `vite.config.js` 中的端口 |

### 7.3 AI 无响应

| 症状 | 原因 | 解决 |
|------|------|------|
| 回答显示 `[LLM未配置]` | API Key 错误 | 检查 `.env` 中的 `DEEPSEEK_API_KEY` |
| 长时间无响应 | 网络问题 | 检查是否能访问 `api.deepseek.com` |
| 回答与景区无关 | 知识库无相关数据 | 上传景区相关文档到知识库 |
| PC/移动端切换不生效 | localStorage 被禁用 | 检查浏览器隐私设置，允许本地存储 |

### 7.4 语音问题

| 症状 | 原因 | 解决 |
|------|------|------|
| 语音输入无反应 | 浏览器不支持 | 使用 Chrome/Edge；降级为文字输入 |
| TTS 语音不播放 | 浏览器禁止自动播放 | 首次需手动点击 🔊 按钮触发 |
| 语音听不到 | 扬声器/音量问题 | 检查系统音量；TTS 音频文件缓存在 `tts_cache/` |

### 7.5 VTube Studio 连接失败

| 症状 | 原因 | 解决 |
|------|------|------|
| 连接超时 | VTS 未启动或 API 未开启 | 启动 VTS → 设置 → 勾选 Start API |
| 认证失败 | 首次连接未确认 | 查看 VTS 弹窗 → 点击允许 |
| `websocket-client` 导入错误 | 依赖未安装 | `pip install websocket-client` |

### 7.6 常用调试命令

```bash
# 检查后端健康
curl http://localhost:8000/api/health

# 测试问答（非流式）
curl -X POST http://localhost:8000/api/chat/ask \

curl http://localhost:8000/api/admin/analytics
```

## 八、生产部署建议

### 8.1 使用生产服务器

```bash
# 替换 uvicorn 开发模式为生产模式
cd backend
python -m uvicorn main:app --host 0.0.0.0 --port 8000 --workers 4 --no-reload
```

### 8.2 前端静态部署

```bash
cd frontend
npx vite build
# 将 dist/ 目录部署到 Nginx/Apache/Caddy
```

**Nginx 配置示例**:
```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    root /path/to/A5/frontend/dist;
    index index.html;

    # API 代理到后端
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # SPA 路由回退
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

### 8.3 安全建议

- `.env` 文件不要提交到版本控制
- 生产环境使用 HTTPS
- 已内置 API 速率限制（默认 30次/分钟/IP，可在 `middleware.py` 中调整）
- 已内置管理后台鉴权（写操作需 `X-Admin-Key` 请求头，默认 Key 在 `.env` 中配置）
